1.에센셜 파일 경로

data 폴더에 담긴것은 data에 plugin 폴더에 담긴것은 plugin에 처럼 맞춰서 넣어주시면 됩니다.

2.각 파일 사용법 및 기능


essential_krweapon.txt
-> 기능
- 이 파일은 다른 플레이어의 인벤보기를 할때 아이템코드를 한글로 번역해서 나타나게 해주는 파일입니다. 
이 파일을 수정할시 인벤보기에 나오는 아이템이름이 바뀝니다.

essential_permission.txt
-> 기능
- 이 텍스트 파일에 해당 플레이어의 스팀 고유 번호를 넣으면 해당 대상은 어드민 권한이 아니여도 어드민 메뉴가 열리며
특정 메뉴 < 어드민 설정 ( 임시 어드민,또는 어드민 추가 삭제 기능, 오프셋 보기 ) > 를 사용 하실수 있습니다.

essential_weaponmenu.txt
-> 기능
- 여기서 무기 메뉴를 불러와서 게임 내부에서 해당 플레이어에게 무기를 줄수 있습니다.
	
essential_setting.txt
-> 기능
- essential.cfg 파일을 대체한 파일입니다. 에센셜에 대한 설정이 들어있으며 기존의 cfg 파일은 인게임에서 유저메뉴를 설정해도
맵이 바뀌면 적용이 풀리는 반면에 이건 항상 적용됩니다. 주로 에센셜에 관한 설정 내용이 다 여기 있습니다.

essential_weaponban.txt
-> 기능
- 서버에서 금지무기로 설정할지 안할지의 관한 텍스쳐입니다. 인게임에서 어드민메뉴에 인벤에디터에서 설정하실수 있습니다.

essential_developmod.txt
-> 기능
- 당신이 플러그인 개발자가 아니라면 굳이 이 기능은 필요가 없습니다. 이 기능은 해당 오프셋값을 인게임에서 즉시 수정하고 보여줄수있는 기능이며
target_offset은 다음에 추가할 예정입니다.

essential_setting.txt 사용법
"settings"
{
	"menucontrol"
	{
		"user_tool"		"1"
		"user_sethome"		"1"
		"user_sendtochat"		"1"
		"user_invsee"		"1"
		"user_teleport"		"1"
	}
	"essential"
	{
		"chat_function"		"1"
	}
}
파일 내부는 이렇게 적혀 있을껍니다.
1은 나는 이기능을 쓰겠다 ( = true )라는것을 의미하며 0은 이 기능은 쓰지 않겠다 ( = false)라는 것을 의미 합니다.
인게임 내부에서도 설정 가능하니 정 이 기능을 모르겠다 하면 게임 내부에서 f3 누르셔서 설정하시면 됩니다.


essential_weaponban.txt 사용법
"weapon_block"
{
	"fa_glock17"
	{
		"ban"		"0"
		"weight"		"90"
	}

파일 내부는 대충 이렇게 적혀있습니다. ban은 무기가 금지무기로 되어있는지 나타내는것이고 weight는 해당 무기의 무게입니다.
이걸 잘못건드리시면 중량쪽에 이상한 문제가 생기실수 있습니다. 6줄 윗칸에서 말했던것처럼 1은 ( = true ) 0은 ( = false )의 의미를 가집니다.

essential_developmod.txt 사용법

"viewoffset"
{
	"my_offset"
	{
		"_vaccinated"
		{
			"name" "백신"
			"type" "int"
		}
	}
	
	"target_offset"
	{
	}
}

이렇게 적혀있을텐데 저기 _vaccinated 이건 오프셋 해당 이름이며 name은 자유로 하셔도 상관없고 type은 아직 int, float 밖에 없습니다. 오프셋 타입에 맞추셔서
쓰시면 됩니다. target_offset은 아직 준비중이며 GetEntProp, GetEntPropFlat 현재는 이것만 지원됩니다.
